# Баг 1: Кнопка "Подтвердить" не работает на экране ввода кода из SMS

## Описание
На экране ввода кода из SMS кнопка "Подтвердить" остаётся неактивной (disabled) даже после того, как пользователь ввёл 4 цифры кода. Кнопка начинает работать только после нажатия на "Отправить код повторно".

## Причина
В `lib/features/auth/widgets/otp_widget.dart` проверка `_codeController.text.length < 4` для определения disabled-состояния кнопки выполняется только в методе `build()`. Но `TextEditingController` не вызывает `setState()` при изменении текста, поэтому виджет не перестраивается, и кнопка остаётся disabled.

Когда пользователь нажимает "Отправить код повторно", `AuthBloc` эмитит новый стейт `otpSent`, что триггерит перестройку `OtpWidget` через `BlocConsumer` в `auth_screen.dart`. При перестройке `build()` вызывается снова, и теперь `_codeController.text.length == 4` — кнопка становится активной.

## Решение
Добавить listener на `_codeController` в `initState`, который вызывает `setState(() {})` при каждом изменении текста. Это обеспечит перестройку виджета и корректное обновление disabled-состояния кнопки.

Файл: `lib/features/auth/widgets/otp_widget.dart`
