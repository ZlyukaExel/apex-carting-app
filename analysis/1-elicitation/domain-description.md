# Описание домена и ограничения

## Описание домена

**Бизнес.** «Апекс» — уличный картинг-центр на окраине города, работающий третий год. Центр организует **групповые заезды** на картах по трассе. Заезд длится 15–20 минут плюс инструктаж перед стартом и проходит на одной из **конфигураций трассы**: короткая — для новичков, длинная — для опытных. Каждый заезд ведёт **маршал-инструктор** (сейчас их пятеро, летом нанимаются подработчики).

**Процесс.** Владелец заранее формирует **расписание на неделю вперёд**: для каждого заезда задаются дата и время старта, конфигурация трассы, назначенный маршал и число картов. Центр располагает **14 картами**; часть картов имеет ограничение скорости. Для новичковых заездов берут не более 8 человек (тяжелее уследить), для опытных — до 14 (все карты). Клиент видит свободные **слоты** заездов и **записывается**, указывая, приедет он со своей экипировкой (шлем, подшлемник) или возьмёт **напрокат**. До заезда клиент может **отменить** запись — при ранней отмене место освобождается, поздняя отмена оставляет карт простаивать. Сейчас запись клиентов ведётся вручную (Telegram + доска маркером), что приводит к двойным записям и путанице, особенно в выходные.

**Цель клиентского приложения.** Дать клиентам самостоятельную онлайн-запись на заезды (просмотр и фильтрация слотов, бронирование, выбор экипировки, отмена, свои бронирования), исключив ручную путаницу и двойные брони. Управление расписанием и работу маршалов/владельца обеспечивает **существующая инфраструктура**; приложение потребляет её данные через API.

**Ключевые объекты домена:**

- **Заезд / слот** — конкретный групповой заезд: дата/время старта, конфигурация трассы, маршал, всего и свободно картов. Создаётся в существующей инфраструктуре; клиенту доступен только для чтения.
- **Конфигурация трассы** — вариант трассы (новичковая / опытная), определяет длительность и потолок картов (справочные данные, read-only).
- **Маршал** — ведёт заезд, проводит брифинг, следит за трассой; для клиента — справочные данные слота (read-only).
- **Карт** — гоночный kart (всего 14); место в заезде = карт.
- **Экипировка** — шлем и подшлемник; может выдаваться напрокат или быть своей.
- **Запись (бронь)** — резерв места клиентом на конкретный слот: число мест, своя/прокатная экипировка, статус.
- **Клиент** — записывается на заезды, отменяет запись.

## Пользователи и роли

Скоуп текущей работы — **только клиентское мобильное приложение и API для него.**
Роли Маршал и Владелец/Администратор обслуживаются **существующей инфраструктурой**
(бэкенд, расписание, админка) и в этот скоуп не входят.

| Роль | Описание | Цели | В скоупе |
| :-- | :-- | :-- | :-- |
| **Клиент** | Гость центра, катается на картах. Пользуется с телефона. | Увидеть и отфильтровать свободные слоты, записаться (себя и гостей), выбрать свою/прокатную экипировку, видеть свои бронирования, отменить запись, получать напоминания. | **Да** |

Маршал и Владелец/Администратор взаимодействуют с системой через **уже реализованные**
интерфейсы существующей инфраструктуры. Клиентское приложение лишь потребляет подготовленные
ими данные (слоты, конфигурации трасс, маршалы) через API.

## Ограничения

**Бизнес-ограничения**

- Всего **14 картов** — общий потолк инвентаря.
- Новичковая группа — **не более 8 человек**; опытная — до **14** (все карты).
- Часть картов имеет ограничение скорости — деталь работы маршалов, в клиентском сценарии не участвует.
- Лимит мест слота = `min(свободные карты, потолок конфигурации трассы)`.
- **Политика отмен:** бесплатная отмена не позднее чем за **2 часа** до старта; поздняя отмена (<2 ч) фиксируется без освобождения места и без денежных штрафов.
- **Оплата офлайн** (наличные / перевод на карту) — приложение лишь фиксирует запись и показывает цену.
- **Период списка слотов по умолчанию — 7 дней (R-027).** По умолчанию клиенту показываются слоты на **ближайшие 7 дней** от текущего момента. Более длинный период доступен через **фильтр дат**. При отсутствии расписания на ближайшие дни показывается **empty state** («Пока нет доступных заездов»).
- **Отмена заезда центром / по погоде (R-008).** Если заезд отменяет центр (организатор или из-за погоды), бронь на стороне клиента **не удаляется**, а получает статус **«Отменён центром»** с указанием причины; клиенту приходит **push-уведомление**. **Повторная запись на отменённый слот запрещена.** Само действие отмены слота выполняется в существующей инфраструктуре (вне скоупа клиентского приложения).

**Технические / платформенные**

- **Mobile-first**: клиентское приложение рассчитано на телефон (использование у трассы).
- Клиентское приложение взаимодействует с **существующим бэкендом через API**; слоты, конфигурации трасс и маршалы доступны только для чтения.
- **Граница серверной интеграции (R-004).** Детали серверной реализации — транзакционность, SLA, маппинг сущностей на внутренние модели бэкенда — **вне скоупа клиентского приложения**. Существующий бэкенд — **black-box источник истины**: гарантия «0 двойных броней» обеспечивается на его стороне (атомарная проверка свободных картов при бронировании). Со стороны клиента граница закрывается **контрактом API**.
- **Данные существующей инфраструктуры (R-015).** Это **учебный/тестовый проект**, исторических данных и легаси-схемы нет. **Канонической схемой считается контракт API**: бэкенд по условию отдаёт ровно те поля, что описаны в контракте. Миграция/backfill и стратегия дозаполнения исторических слотов вне скоупа.

**Проект**

- **Срок** — к старту высокого сезона, ~2 месяца (жёсткая цель).
- **Бюджет** ограничен → начинаем с самого нужного, «навороты» позже.

**Границы скоупа**

Скоуп — **клиентское мобильное приложение и API для него**. Управление расписанием, роли маршала и владельца, оценки, админка — это **существующая инфраструктура**, вне скоупа.

| В скоупе клиентского приложения | Вне скоупа (существующая инфраструктура / не делаем сейчас) |
| :-- | :-- |
| Регистрация/вход клиента по имени и телефону | Роли Маршала и Владельца/Администратора, админ-экран |
| Просмотр и **фильтрация** списка слотов (дата, тип трассы, наличие мест, маршал) | Создание и редактирование расписания (слоты приходят из существующего бэкенда через API) |
| Самостоятельная запись, в т.ч. несколько мест за раз | Оценка маршала и рейтинги — **Phase 2** |
| Выбор «своя / прокатная экипировка», контроль лимита мест и прокатного фонда | Отметка явки/неявки, отмена слота маршалом + массовое уведомление |
| Список своих бронирований | Онлайн-оплата; ручная отметка «оплачено» |
| Отмена записи с правилом 2 часов | Автоматический учёт погоды по прогнозу |
| Цена и фиксация записи (оплата офлайн) | Программа лояльности для постоянных клиентов |
| Напоминания о записи — по возможности в срок | Публичный показ рейтинга клиентам — **Phase 2** |

## Глоссарий

| Термин | Определение |
| :-- | :-- |
| **Заезд / слот** | Запланированный групповой заезд: дата/время старта, конфигурация трассы, маршал, всего и свободно картов. |
| **Конфигурация трассы** | Вариант трассы: новичковая (короткая, потолок 8) или опытная (длинная, до 14). |
| **Группа** | Состав клиентов, записанных на один слот. |
| **Карт** | Карт из гаража центра; всего 14. |
| **Экипировка** | Шлем и подшлемник; может выдаваться напрокат или быть своей. |
| **Своя экипировка** | Клиент приезжает со своим шлемом и подшлемником; занимает место в заезде, но не прокатный фонд. |
| **Запись (бронь)** | Резерв одного или нескольких мест на конкретный слот: число мест, своя/прокатная экипировка, статус. |
| **Отмена** | Снятие записи. Клиентом: ранняя (≥2 ч до старта) освобождает место; поздняя (<2 ч) фиксируется без освобождения места. Центром (организатором / по погоде): бронь переходит в статус «Отменён центром» с причиной, клиент получает push, повторная запись на этот слот запрещена. |
| **Отмена центром** | Отмена заезда организатором или из-за погоды (выполняется в существующей инфраструктуре). На стороне клиента бронь сохраняется со статусом «Отменён центром» + причина; отправляется push-уведомление; повторная запись на отменённый слот недоступна. |
| **Администратор / владелец** | Денис — управляет расписанием и маршалами через существующую инфраструктуру (вне скоупа клиентского приложения). |
| **MVP** | Минимально жизнеспособный продукт: набор функций к старту высокого сезона. |
