# Apex Картинг-клуб — мобильное приложение

Мобильное приложение для бронирования заездов в картинг-клубе **«Apex»**.  
Разработано в рамках учебного проекта.

## Структура проекта

```
carting-club/
├── analysis/          # Аналитика и требования
├── apex_app/          # Мобильное приложение
└── debug_report/      # Баги, найденные и исправленные
```

## Требования для сборки

- **Flutter SDK** 3.18+ (Dart 3.9.2+)
- **Android Studio**
- Эмулятор Android или физическое устройство

## Инструкция по сборке

Все команды выполняются из директории `apex_app/`:

```bash
# Установить зависимости
flutter pub get

# Запустить в режиме отладки
flutter run

# Собрать APK (Android)
flutter build apk

# Запустить тесты
flutter test
```

Подключение к реальному бэкенду не требуется — приложение работает на мок-данных.

### Использованные промпты

1. **Анализ**
К сожалению, промпт не сохранился.

2. **Реализация** 
Твоя задача - реализовать любые 3 фичи из анализа/ТЗ. Для этого создай простое Flutter приложение для Андроид. Сложный интерфейс не нужен. Комментарии пиши на русском, за исключением названий и жаргона. При наличии каких-либо вопросов, задавай их мне, я буду выбирать, что делать. Так как сервера у нас нет, запросы просто имитируй.

3. **Тестирование**
Ты дебажишь приложение в папке apex_app. Создай в текущей директории папку debug_report, туда ты будешь писать отчеты о дебаге. Краткое описание проблемы, причина и решение, каждый отчет в отдельном файле. Пиши на русском, названия, жаргон и комментарии в коде можно не переводить. Текущие баги: 1. На экране ввода кода из смс кнопка "Подтвердить" не работает пока не нажмешь кнопку "Отправить код повтроно". 2. Если введен неверный код из смс, пользователя перебрасывает обратно на экран с вводом номера телефона. 3. На главном экране "Апекс - заезды" Debug сборка на Android показывает ошибку "fit overflowed by 23 pixels" в разделе с фильтрами "Сегодня", "Неделя", "Выходные" и тд. Также создай несколько тест-кейсов.
